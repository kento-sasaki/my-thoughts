# AI 時代の開発・ワークフロー観

## これは何？

AI で実装の表層が安くなった世界で、エンジニアの仕事の重心はどこへ移るのか。ワークフローと判断のかたちについての考えをまとめる。

## 通底する考え

AI が変えたのは「作る速さ」だけではない。**何に価値があるか**を変えた。表層を量産することの価値が下がり、相対的に「何を作るかを決めること」と「壊れない構造を用意すること」の価値が上がった。だから、表現の量産は AI に任せて、人間とコードは**決定と構造**に集中する方向へ寄せていきたいと思っている。

この章では、それを2つの面から書く。

- [表層から設計へ](./01_表層から設計へ.md) — 人間側の重心。表層実装が安くなると、仕事は「設計を決めること」へ移る。
- [ドキュメントより実行可能な制約へ](./02_実行可能な制約へ.md) — AI への渡し方。ルールは「読ませる文章」より「実行できる制約」にするほうが効く。
- [LLM には厳格な制約を](./03_LLMには厳格な制約を.md) — 人間に窮屈な制約ほど LLM には足場になる。制約は実行可能な検証ループの供給源であり、人間の重心は「正しさを定義し判断する」層へ移る。
