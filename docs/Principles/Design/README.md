# UI・デザイン観と技術倫理

## これは何？

なぜエンジニアがデザインについて考える必要があるのか。UI を「認知のフレーム」として捉える視点と、道具を作る側としての技術倫理についての考えをまとめる。

## 通底する考え

私は、UI を「見た目をどう整えるか」の話だとは思っていない。UI は、ユーザーが何を期待し、何ができ、何に注意を向けるかを決めている。いわば**認知と行動のフレーム**だ。つまり作り手は、画面を作りながら、知らず知らずユーザーの行動を形作っている。

「我々は道具を形作り、道具は我々を形作る」。この章はこの一文を背骨にしている。[UI は認知フレームの設計](./01_UIは認知フレーム.md)はユーザーの期待をどう尊重するかの話、[ユーザーの慣れという、見えない制約](./03_慣れという制約.md)はその期待＝「慣れ」をどう読み、注意という有限の資源をどう扱うかの話、そして[道具が我々を形作る](./02_道具が我々を形作る.md)はそれらの力に作り手としてどう責任を持つかの話だ。

エンジニアがデザインを考えるのは、美的センスのためではない。自分が作るものが人の認知と時間に影響を与えている、という事実から目を逸らさないためだ。
