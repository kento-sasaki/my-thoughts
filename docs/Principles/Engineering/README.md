# 設計の原則

## これは何？

コンポーネントやモジュールをどう分け、どう繋ぐか。設計判断の根拠となる原則をまとめる。
個別の[コード規約](../../Practices/StyleGuide/README.md)が「何をすべきか（what）」を書き、この章は「なぜそうするのか（why）」を書く。
