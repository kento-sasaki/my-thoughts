# 不要な型キャストを避け型ガードを使う

## 規約

- `as` による型キャストではなく、型ガード関数で型を絞り込む。
- 実際には `null` にならない値に不要な `| null` を付けない（non-null assertion `!` を不要にする）。

```ts
// 👎 as でコンパイラを欺く / 不要な | null で ! が必要になる
const token = currentUser?.jwt as string | undefined
const result = props.value! // value: string | null

// 👍 型ガードで絞り込む / 型から null を除外する
const token = currentUser && (isDoctorUser(currentUser) || isPatientUser(currentUser)) ? currentUser.jwt : undefined
const result = props.value   // value: string
```

`as` は実行時の安全性を保証しない。型ガードを使えばコンパイラが実行パスを追跡でき、実行時にも安全になる。不要な `| null` を避ければ型の正確性が上がる。
