# 不要な型キャストを避け型ガードを使う

## 概要

- `as` による型キャストではなく、型ガード関数を使って型を絞り込む。
- 実際には `null` にならない値に不要な `| null` を付けない（non-null assertion `!` を不要にする）。

## 👎 不適切な例

```ts
// as で型キャストしてコンパイラを欺く
const token = currentUser?.accessToken as string | undefined

// 不要な | null で non-null assertion が必要になる
type Props = {
  value: string | null
}

const result = props.value!
```

## 👍 適切な例

```ts
// 型ガード関数で型を絞り込む
const token = currentUser
  ? isAdminUser(currentUser) || isMemberUser(currentUser)
    ? currentUser.accessToken
    : undefined
  : undefined

// 実際に null になり得ないなら型から除外する
type Props = {
  value: string
}
const result = props.value
```

## なぜこのルールが必要か

`as` による型キャストはコンパイラを欺くだけで、実行時の安全性を保証しない。型ガード関数を使えばコンパイラが実行パスを追跡でき、実行時にも安全になる。また、不要な `| null` を避けることで non-null assertion (`!`) が不要になり、型の正確性が上がる。
