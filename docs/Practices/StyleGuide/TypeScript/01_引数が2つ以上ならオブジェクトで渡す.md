# 関数の引数が2つ以上の場合はオブジェクトで渡す

## 概要

- 引数が2つ以上ある関数は、位置引数ではなくオブジェクトで受け取る。

## 👎 不適切な例

```ts
// 位置引数だと、呼び出し側でどの引数が何か分からず、順序ミスも起きやすい
const generateChartData = (values: number[], yAxis: number[], xAxis: Date[]) => { /* ... */ }
const data = generateChartData(values, yAxis, xAxis)
```

## 👍 適切な例

```ts
// オブジェクトで渡して展開する
const generateChartData = ({ values, yAxis, xAxis }: { values: number[]; yAxis: number[]; xAxis: Date[] }) => { /* ... */ }
const data = generateChartData({ values, yAxis, xAxis })
```

## なぜこのルールが必要か

位置引数では呼び出し側で各引数の意味が見えず、同じ型の引数が並ぶと順序を取り違えても型エラーにならない。オブジェクトで渡せば呼び出し側で各引数の意味が名前で明示され、順序ミスもなくなる。引数の追加・削除にも強く、デフォルト値や省略可能な引数も扱いやすい。

## 例外

特になし。
