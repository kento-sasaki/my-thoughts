# フロントエンドの命名規則

## コンポーネント・型

- コンポーネント・型はパスカルケースの名詞（`Button`、`ColorPalette`）。
- props の型は `Props` または具体名（`ButtonProps`）。
- レイヤを名前に込める：`{Feature}Page` / `{Feature}Cell` / `{Feature}View`（ファイルは `{Feature}.view.tsx`）。

## 関数・ハンドラ

- 関数はキャメルケースの動詞（`handleSubmit`）。
- props で受け取るコールバックは `on + 動詞 + 目的語`（`onCompleteLearning`）。
- コンポーネント内で定義するハンドラは `handle + 動詞 + 目的語`（`handleSubmit`）。
- Custom Hooks は `use + 機能名`（`usePatientSignUp`）。

## GraphQL

- Query：`Find + エンティティ + Query`、Mutation：`Create/Update/Delete + エンティティ + Mutation`。

## 避けること

- 略語の多用、日本語のローマ字表記、曖昧な名前（`data` / `info` / `item`）。

→ なぜ：[命名は言語](../../../Principles/Engineering/02_命名は言語.md)
