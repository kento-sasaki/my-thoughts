# 親に依存するコンポーネントを作らない

## 規約

- 親が特定の実装をしている前提のコンポーネントを作らない。
- 例：親が `<ul>`（`List.Root`）であることを期待して `<li>`（`List.Item`）だけを返すコンポーネントを作らない。

```tsx
// 👎 親が List.Root であることを知っている必要がある
export const ListItem: FC<{ label: string }> = ({ label }) => <List.Item>{label}</List.Item>

// 👍 親が何であれ単体で完結する
export const List: FC<{ items: string[] }> = ({ items }) => (
  <List.Root>{items.map((i, k) => <List.Item key={k}>{i}</List.Item>)}</List.Root>
)
```

前者は親と子に余計な依存が生まれる。子は自分の構造を自分で完結させる。
