# 親に依存するコンポーネントを作らない

## 概要

- 親が特定の実装をしていることを前提としたコンポーネントを作らない。
    - 例えば、親が `<ul>` であることを期待して、 `<li>` だけのコンポーネントを作るべきではない。

## 👎 不適切な例

```tsx
// ul にラップされる前提になっており、
// このコンポーネントは親側の実装を知っている必要がある
export const ListItem: FC<{ label: string }> = ({ label }) => {
  return (
    <li>{label}</li>
  )
}
```

## 👍 適切な例

```tsx
// このコンポーネントは親が何であれこれ単体で機能でき、余計な依存がない
export const List: FC<{ items: string[] }> = ({ items }) => {
  return (
    <ul>
      {items.map((item, index) => (
        <li key={index}>{item}</li>
      ))}
    </ul>
  )
}
```

## なぜこのルールが必要か

前者の例のようなコンポーネントを作った場合、親側からすると `ListItem` の実装を知っていて適切なコンポーネントでラップしなければいけなくなり、`ListItem` からすると、自身を呼ぶ親コンポーネントは `ul` を使っている前提で実装していることになる。
いずれにせよこれは親と子に余計な依存が生じている実装になっている。子は自分の構造を自分で完結させるべきである。

→ なぜ：[最小依存性：親に依存しないコンポーネント](../../../Principles/Engineering/01_最小依存性.md)
