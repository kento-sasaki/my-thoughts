# フロントエンドのディレクトリ構成

> このノートは個々の規約スニペットではなく、ディレクトリ構成の参照ドキュメント。`不適切な例 / 適切な例` 形式ではなく構成図でまとめる。

## 概要

- 共通コンポーネントは Atomic Design（`atoms` / `molecules` / `organisms`）で分類し、各コンポーネントはディレクトリを切って `{Name}.tsx` / `.stories.tsx` / `.test.tsx` を同居させる。
- 画面は `pages/{Role}/{Feature}Page/` に置き、Page 配下に `components/`（`{Feature}Cell` / `{Feature}.view.tsx` / `hooks/` / `contents/`）を構成する。

## ディレクトリ構造

```
web
└── src
    ├── components
    │   ├── atoms
    │   │   └── {ComponentName}
    │   │       ├── {ComponentName}.tsx
    │   │       ├── {ComponentName}.stories.tsx
    │   │       └── {ComponentName}.test.tsx
    │   ├── molecules
    │   │   └── {ComponentName}
    │   │       └── （atoms と同様）
    │   └── organisms
    │       └── {ComponentName}
    │           └── （atoms と同様）
    └── pages
        ├── User
        │   └── {FeatureName}Page
        │       ├── {FeatureName}Page.tsx
        │       ├── {FeatureName}Page.stories.tsx
        │       ├── {FeatureName}Page.test.tsx
        │       └── components
        │           ├── {FeatureName}Cell
        │           │   ├── {FeatureName}Cell.tsx
        │           │   ├── {FeatureName}Cell.stories.tsx
        │           │   ├── {FeatureName}Cell.test.tsx
        │           │   └── {FeatureName}Cell.mock.ts
        │           ├── {FeatureName}
        │           │   ├── {FeatureName}.view.tsx
        │           │   └── {FeatureName}.view.stories.tsx
        │           ├── hooks
        │           │   └── use{FeatureName}.ts
        │           └── contents
        │               └── {featureName}Contents.tsx
        └── Admin
            └── ...
```

関連ファイル（実装・Story・テスト・モック）をコンポーネント単位で同居させ、見つけやすさと凝集度を保つ。命名は[フロントエンドの命名規則](./04_命名規則.md)に従う。
