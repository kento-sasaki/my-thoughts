# フロントエンドのディレクトリ構成

## 規約

- 共通コンポーネントは Atomic Design（`atoms` / `molecules` / `organisms`）で分類し、各コンポーネントはディレクトリを切って `{Name}.tsx` / `.stories.tsx` / `.test.tsx` を同居させる。
- 画面は `pages/{Role}/{Feature}Page/` に置き、Page 配下に `components/`（`{Feature}Cell` / `{Feature}.view.tsx` / `hooks/` / `contents/`）を構成する。

```
web/src
├── components/{atoms,molecules,organisms}/{Component}/
│     {Component}.tsx / .stories.tsx / .test.tsx
└── pages/{Role}/{Feature}Page/
      {Feature}Page.tsx / .stories.tsx / .test.tsx
      └── components/
            ├── {Feature}Cell/   （.tsx / .stories.tsx / .test.tsx / .mock.ts）
            ├── {Feature}/        （.view.tsx / .view.stories.tsx）
            ├── hooks/use{Feature}.ts
            └── contents/{feature}Contents.tsx
```

関連ファイル（実装・Story・テスト・モック）をコンポーネント単位で同居させ、見つけやすさと凝集度を保つ。命名は[フロントエンドの命名規則](./04_命名規則.md)に従う。
